# Subscription-Churn-warning-system

## Project Overview

Customer churn is one of the biggest challenges for subscription-based businesses.
This project builds a SQL-based early warning system to identify customers likely to churn before cancellation, enabling proactive retention strategies.

The analysis is implemented entirely in MySQL (SQL only), focusing on business logic, window functions, and modular SQL views.

## Key Features
- Identifies churn risk using user activity and payment behavior
- Assigns a churn risk score to each customer
- Classifies users into High, Medium, and Low risk categories
- Designed for real-world subscription businesses

## Tools Used
- MySQL 8.0
